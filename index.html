Git 사용법

이 내용은 git bash를 이용하는 방법이며, 실제로는 Vscode를 사용한다.

예제에 사용하는 파일

  1. index.html
  2. main.css
  3. main.js
  4. readme.md (markdown 문서)
  5. .gitignore
  6. log.txt

순서

  • 해당 디렉터리로 이동하여

git init

  • git add 명령으로 추적할 파일을 추가(add)한다.

git add index.html

  • add 시킨 파일을 ‘ rm –cached’를 사용하여 추적(track)에서 제외시킨다. wildcard(asterisk) 문자를 사용할 수 있다. 모든 파일을 add하려면
    (‘ git add . ’)

git rm –cached index.html

  • 모든 파일을 add 시킨다.

git add .

  • git status 로 상태를 확인한다.

git status

  • git commit 명령으로 vim editor로 들어간다.

git commit

  • Vim editor 상태
  1. ’ i ’를 입력하여 insert mode로 들어간다.
  2. commit할 문자를 입력한다. (commit할 문자란 github에 내가 어떤 내용을 커밋했는지 상기시키도록 해줄 내용이다.)

  3. 입력이 끝나면 esc key를 눌러 insert mode에서 나온다.

  4. :wq 를 입력하고 enter.

  5. vim editor 상태에서 빠져 나온다.

  • 문서를 수정한 다음 ’수정내용(commit message)’을 커밋할 경우

    • 특정 파일(예를 들어 index.html)을 커밋할 경우 > git add index.html > git commit -m ’changed index.html

    여기서 -m은 ’message’란 의미고, 따옴표의 ’changed index.html’은 커밋 메시지의 내용이다.

  • Git에서 관리하지 않을 문서를 지정(git ignore)하는 방법

    • 특정 파일(예를 들어 index.html)을 커밋할 경우

    git add index.html

    git commit -m ’changed index.html

    여기서 -m은 ’message’란 의미고, 따옴표의 ’changed index.html’은 커밋 메시지의 내용이다.
